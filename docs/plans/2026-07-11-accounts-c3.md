# Accounts: cross-account authz (C3) + DISTRICT/CENTRAL collapse — Implementation Plan (3b of 4)

>
> **⚠️ This plan touches `internal/issuance` — the fail-closed credential trust chain. Task 6 (the cabinet-confinement test) is a HARD GATE: it must pass before the issuance change in Task 7 is considered safe. Golden byte-comparison will NOT catch a confinement bug; the confinement test is the only thing that will.**

**Goal:** Close **C3** (cross-account JetStream sourcing authz is unmodeled) and **D6** (account model), by collapsing `DISTRICT_<d>`+`CENTRAL` into one `INTERNAL` account (Decision 6) and generating the `$JS.API` service export/import pairs the DMZ (and future PEER) needs to source across the account boundary — with the renamed API prefix the reference tests proved is required.

**Architecture:** Decision 6 makes cabinet→regional→central **intra-account** (no service imports on the hot path), leaving only the **DMZ egress** cross-account. `INTERNAL` exports `$JS.<core-domain>.API` (service) + a `deliver` subject; `DMZ` imports both (renamed to `JS.INTERNAL.API`); the DMZ stream's source uses that renamed prefix + a deliver prefix. Cabinets move from `DISTRICT_<d>` to `INTERNAL` but stay **subject-scoped to their district** (their ACL is unchanged; only the account they live in changes). The acceptance spec is the 4 merged reference tests in `test/integration/dmz_flow_test.go`.

**Tech Stack:** Go, `internal/accounts` (model), `internal/render/accounts.tmpl`, `internal/plan` (IR `Source`), `internal/render/{k8s,baremetal}.go`, `internal/issuance` (`mintCabinets`), `internal/naming`, golden trees, embedded-NATS integration + a confinement test.

## Global Constraints

- **Reference config (proven, `test/integration/dmz_flow_test.go`):** origin account exports `{ service: "$JS.<domain>.API.>" }` + `{ stream: "deliver.<x>.>" }`; consumer imports `{ service: { account: <origin>, subject: "$JS.<domain>.API.>" }, to: "JS.<origin>.API.>" }` + `{ stream: { account: <origin>, subject: "deliver.<x>.>" } }`; the source uses `External{ APIPrefix: "JS.<origin>.API", DeliverPrefix: "deliver.<x>" }`. **A no-rename import does not deliver — the rename is mandatory.**
- **Naming SSOT:** internal account name + the `JS.<account>.API` rename + the deliver subject are new `internal/naming` helpers. Never inline them (`cmd/issue` must agree).
- **Confinement invariant (fail-closed):** a minted cabinet cred must be able to pub/sub **only** its district subject space and be **denied** everywhere else (esp. `$JS.API.>` and other districts). This holds before AND after the collapse. Task 6 pins it.
- **Golden protocol:** `make golden` + review. The collapse rewrites `accounts.conf` (one INTERNAL account replaces N DISTRICT_<d> + CENTRAL) and the issuance goldens (account key set changes). DMZ stream `externalApiPrefix` becomes the renamed prefix; other streams unchanged. Review every diff.
